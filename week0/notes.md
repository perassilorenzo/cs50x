# CS50x - Week 0: Scratch

## Computational Thinking

Computer Science is fundamentally about **computational thinking** and **problem solving**.

The goal is to take a complex problem, break it into smaller parts, and create a solution that a computer can execute efficiently.

---

## Binary

Computers work using the **binary system**, which consists of only two digits:

* `0`
* `1`

A single binary digit is called a **bit**.

### Decimal vs Binary

Humans typically use the decimal system (base 10), while computers use binary (base 2).

Example:

```text
Decimal: 65
Binary: 01000001
```

---

## Bytes and ASCII

Eight bits form one **byte**.

```text
8 bits = 1 byte
```

Computers can represent characters using encoding systems such as ASCII.

Examples:

```text
A = 01000001 = 65
H = 72
I = 73
! = 33
```

Therefore:

```text
01001000 01001001 00100001
```

represents:

```text
HI!
```

### Uppercase and Lowercase Letters

ASCII assigns different values to uppercase and lowercase characters.

```text
A = 65
a = 97
```

The difference is:

```text
97 - 65 = 32
```

---

## Colors and RGB

Digital colors are commonly represented using the RGB model:

* Red
* Green
* Blue

Each channel can have a value from 0 to 255.

Examples:

```text
0   0   0     = Black
255 255 255   = White
```

Each color channel requires 8 bits.

```text
8 + 8 + 8 = 24 bits
```

Therefore:

```text
24 bits = 3 bytes
```

are required to represent a single RGB color.

---

## Representation of Media

Computers represent different types of information using binary data.

### Images

Images are collections of pixels, each represented by color values.

### Videos

Videos are sequences of images displayed rapidly to create motion.

### Audio

Audio can be represented through:

* Frequency
* Duration
* Loudness

---

## Algorithms

An **algorithm** is a step-by-step procedure used to solve a problem.

Different programming languages can implement the same algorithm.

Examples:

* C
* C++
* Java
* Python

Writing software is not only about obtaining the correct result, but also about:

* Speed
* Efficiency
* Memory usage
* Resource consumption
* Cost

A better algorithm often leads to a better solution.

---

## Code

Code is an implementation of an algorithm using a programming language.

Programs generally:

1. Receive input.
2. Process data.
3. Produce output.

---

## Pseudocode

Pseudocode is an informal way of describing the logic of a program before writing actual code.

It helps programmers focus on problem solving rather than syntax.

---

## Functions

Functions are reusable blocks of code that perform specific tasks.

They help organize programs into smaller and more manageable components.

---

## Conditions

Conditions allow programs to make decisions.

They behave like a fork in the road:

```text
If condition is true → do one thing
Else → do another
```

---

## Boolean Expressions

A Boolean expression is a question that can have only two possible answers:

```text
True / False
1 / 0
Yes / No
```

Boolean expressions are commonly used in conditions.

---

## Loops

Loops allow programs to repeat actions automatically.

Instead of writing the same instructions multiple times, a loop executes them repeatedly until a condition changes.

---

## Scratch

Scratch introduces fundamental programming concepts through visual blocks.

Key concepts practiced in Scratch:

* Functions
* Conditions
* Loops
* Variables
* Return values
* Problem decomposition

Scratch focuses on logic and problem solving before introducing programming syntax.
